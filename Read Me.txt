Real-Time Transaction Ledger Engine
A robust backend ledger system built with Node.js, Express, and SQLite that handles synchronous payment data replication and live transaction tracking via the Stripe API and Stripe CLI webhooks.
This system is designed to perform data synchronization upon system boot and gracefully intercept real-time webhooks to manage local state alongside Stripe’s event streaming.

Key Features
Cold-Start DB Synchronization: Automatically fetches, cleans, and updates existing cardholders and card structures from the Stripe API on initial server boot.
Real-Time Data Sync: Leverages webhook endpoints via Stripe CLI tunnels to listen for downstream updates, immediately mapping account states natively to a local SQLite database.
Transaction Authorization Lifecycle: Simulates and handles real-time point-of-sale authorizations, updating dynamic ledger data layouts instantly.

Tech Stack
Runtime Environment: Node.js
Backend Framework: Express.js
Database Engine: SQLite (Embedded relational ledger)
API Integration: Stripe API & Stripe CLI (Webhooks)

Installation & Setup
1. Prerequisites
Ensure you have the Stripe CLI installed and authenticated with your test account.
2. Clone the Repository
Bash
git clone https://github.com/ngabriel/YOUR-REPO-NAME.git
cd YOUR-REPO-NAME
3. Install Dependencies
Bash
npm install
4. Initialize the Webhook Tunnel
In a separate terminal window, open a secure tunnel to forward webhook payloads securely to your local port layout:
Bash
stripe listen --forward-to localhost:3000/webhook
5. Boot the Server
Start your Node server to trigger the initial database boot-sync:
Bash
node server.js

Testing and API Simulation Examples
Create a Live Cardholder
Use the Stripe CLI to provision a new user configuration and witness the real-time SQLite database replication:
Bash
stripe post /v1/issuing/cardholders -d name="Michael Jackson" -d email="joseph.rosen@example.com" -d status=active -d type=individual -d "individual[first_name]"="Michael" -d "individual[last_name]"="Jackson" -d "billing[address][line1]"="123 Innovation Way" -d "billing[address][city]"="London" -d "billing[address][postal_code]"="EC1A 1BB" -d "billing[address][country]"="GB" -d phone_number="+447123434789"
Issue a Virtual Card
Link a physical or virtual payment asset directly to your newly created cardholder id layout:
Bash
stripe post /v1/issuing/cards -d cardholder="YOUR_CARDHOLDER_ID" -d currency=gbp -d type=virtual -d status=active
Simulate an Authorization Swipe
Simulate an incoming synchronous point-of-sale purchase ledger adjustment:
Bash
stripe post /v1/test_helpers/issuing/authorizations -d card="YOUR_CARD_ID" -d amount=1200 -d currency=gbp
